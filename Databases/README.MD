数据库历史的时间表

1960年代

计算机数据库始于1960年代，当时计算机的使用成为私有组织更具成本效益的选择。在这十年中，有两种流行的数据模型：称为CODASYL的网络模型和称为IMS的分层模型。 SABER系统被证明是商业上成功的一种数据库系统，该系统已被IBM用来帮助美国航空管理其预订数据。

1970年至1972年

E.F. Codd发表了一篇重要的论文，提出了使用关系数据库模型的建议，他的想法改变了人们对数据库的思考方式。在他的模型中，数据库的架构或逻辑组织与物理信息存储断开连接，这成为数据库系统的标准原理。

1970年代

在1974年至1977年之间两个主要的关系数据库系统原型被创建，分别是在UBC开发的Ingres和在IBM San Jose创建的SystemR。 Ingres使用一种称为QUEL的查询语言，从而导致创建了诸如Ingres Corp.，MS SQL Server，Sybase，Wang’s PACE和Britton-Lee之类的系统。另一方面，System R使用SEQUEL查询语言，它为SQL / DS，DB2，Allbase，Oracle和Non-Stop SQL的开发做出了贡献。关系数据库管理系统（RDBMS）也在这个十年中成为公认的术语。

1976年

P. Chen今年提出了一个新的数据库模型，称为Entity-Relationship，即ER。该模型使设计人员可以专注于数据应用程序，而不是逻辑表结构。

1980年代

结构化查询语言或SQL成为标准查询语言。计算机销量的快速增长推动了数据库市场的发展，关系数据库系统获得了商业上的成功，这导致网络和分层数据库模型的普及率大大下降。 DB2成为IBM的旗舰数据库产品，并且IBM PC的引入导致许多新数据库公司的成立以及诸如PARADOX，RBASE 5000，RIM，Dbase III和IV，OS / 2 Database Manager，和Watcom SQL。

1990年代初期

在数据库行业的崩溃之后，大多数幸存的公司以高价出售了复杂的数据库产品。大约在这个时候，发布了用于应用程序开发的新客户端工具，其中包括Oracle Developer，PowerBuilder，VB等。还开发了许多个人生产力工具，例如ODBC和Excel / Access。对象数据库管理系统（ODBMS）的原型是在1990年代初创建的。

1990年代中期

互联网的出现导致了数据库行业的指数级增长。普通桌面用户开始使用客户端-服务器数据库系统来访问包含旧数据的计算机系统。

1990年代后期

在线业务投资的增加使得人们对Internet数据库连接器的需求增加，例如Front Page，Active Server Pages，Java Servelets，Dream Weaver，ColdFusion，Enterprise Java Beans和Oracle Developer2000。cgi，gcc，MySQL的使用，Apache和其他系统为Internet带来了开源解决方案。随着销售点技术的日益普及，在线交易处理和在线分析处理开始走向成熟。

2000年代

尽管Internet行业在2000年代初经历了下滑，数据库应用程序仍在继续增长。为销售点交易和供应商合并开发了新的交互式应用程序。目前，世界上三大数据库公司是Microsoft，IBM和Oracle。

今天

如今数据库无处不在，用于提高我们的日常生活。从个人云存储到天气预报，我们今天使用的许多服务背后可能都有数据库的支持。非关系数据库领域中有许多新的参与者提供特定的解决方案，例如当前的一些关系数据库包括Oracle，MySQL和DB2等巨头。当然现在还有致力于使所有人都能使用强大技术的新趋势。 Quick Base是一个建立在关系数据库上的在线数据库平台，它使任何技能水平的用户都可以使用关系数据库的强大功能来创建自定义应用程序，并且操作简单，只需点击一下鼠标即可。
